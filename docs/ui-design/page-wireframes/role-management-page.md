# 项目权限设置页面UI设计

## 1. 页面概述

### 1.1 功能目标
- 设置分支保护规则
- 配置代码所有者(CODEOWNERS)
- 管理团队权限分配
- 简单的权限管理界面

### 1.2 页面定位
- 项目设置的核心功能页面
- 只有仓库所有者可以访问
- 影响项目的安全性和协作流程

## 4. 交互设计

### 4.1 分支保护规则交互
- 点击复选框启用/禁用保护规则
- 分支名称输入框支持添加多个分支
- 高级设置折叠/展开显示
- 实时预览保护规则效果

### 4.2 代码所有者配置交互
- 用户名输入框支持@提及自动完成
- 路径模式支持通配符和目录匹配
- 规则列表支持拖拽排序
- 实时预览CODEOWNERS文件内容

### 4.3 团队权限分配交互
- 团队选择下拉框显示可用团队
- 权限级别选择器
- 团队列表支持编辑和移除
- 批量操作支持

## 6. 状态指示

### 6.1 设置状态
- **已保存**: 绿色勾选，设置已保存到GitHub
- **未保存**: 黄色警告，有未保存的更改
- **错误**: 红色错误，设置保存失败
- **加载中**: 旋转图标，正在保存设置

### 6.2 权限状态
- **已启用**: 绿色圆点，权限规则已启用
- **已禁用**: 灰色圆点，权限规则已禁用
- **冲突**: 红色警告，权限配置有冲突
- **待验证**: 黄色问号，需要验证权限设置

## 7. 功能流程

### 7.1 设置分支保护流程
```
选择分支 → 启用保护规则 → 配置审查要求 → 设置状态检查 → 保存设置
```

### 7.2 配置代码所有者流程
```
添加全局所有者 → 设置特定文件规则 → 预览CODEOWNERS文件 → 保存配置
```

### 7.3 分配团队权限流程
```
选择团队 → 设置权限级别 → 验证权限设置 → 保存分配
```
